# 가구 넘어짐 시뮬레이터 UI 리디자인 계획 (Test Control Flow)

## Goal Description
제공해주신 레퍼런스 이미지(미니멀한 오디오 플러그인 스타일, 고급스러운 대시보드 스타일)를 바탕으로 시뮬레이터의 UI/UX를 전면 개편합니다. 핵심 목표는 사용자가 단순한 시뮬레이션을 보는 것이 아니라, **"정밀한 물리 테스트 장비를 직접 조작하여 가구의 안정성을 평가하는 플로우"**를 경험하게 하는 것입니다. 기존의 투박한 슬라이더나 버튼 대신 시각적으로 만족스럽고 직관적인 '노브(Knob)/다이얼(Dial)' 및 명확한 데이터 게이지를 도입하여 몰입감을 높입니다.

## User Review Required

> [!IMPORTANT]
> 계획 승인 전 다음 사항에 대한 의견을 부탁드립니다.
> 1. 제공해주신 두 이미지 중 어느 쪽에 더 가깝게 전반적인 톤을 조절할까요?
>    - 옵션 A: 첫 번째 이미지처럼 극도로 미니멀한 라이트 그레이 배경 + 다크 챠콜 다이얼 (매우 깔끔한 플러그인 느낌)
>    - 옵션 B: 두 번째 이미지처럼 부드러운 그린 톤이 가미된 반투명(Glassmorphism) 데이터 대시보드 느낌
>    - 또는 두 가지의 장점을 섞어 (미니멀 바탕 + 데이터 시각화에만 부드러운 톤 사용) 진행해도 좋습니다.

## Proposed Changes

### UI/UX Foundation & Design System
UI를 제어 패널(좌/우측 또는 하단)과 메인 테스트 뷰(중앙 3D 캔버스)로 명확히 분리하여 사용자가 실험을 통제하는 느낌을 줍니다.

#### [MODIFY] furniture-sim/index.html
- 전체 레이아웃을 '테스트 컨트롤 센터' 형식(Dashboard Layout)으로 재구성.
- 기존 `<input type="range">` 등의 기본 UI 요소를 커스텀 다이얼/노브 컴포넌트를 담을 컨테이너 렌더링 영역으로 교체.

#### [MODIFY] furniture-sim/style.css
- 레퍼런스 이미지에 기반한 CSS 변수(Variables) 정의: 배경색, 텍스트 색상, 그리고 특히 노브나 패널에 깊이감을 줄 부드러운 그림자(Soft Shadows) 스타일 추가.
- 커스텀 다이얼 및 게이지 UI를 위한 스타일링(둥근 테두리, 부드러운 그라데이션, 포인터 인디케이터 등) 추가.
- 모던하고 깔끔한 타이포그래피(예: Inter 또는 로컬 산세리프 폰트) 적용.

### Custom Control Logic Integration
#### [MODIFY] furniture-sim/main.js (or equivalent JS entry file)
- **커스텀 노브(Knob) 로직 구현**: 마우스 드래그(위/아래 또는 회전)를 통한 값 변경 로직 작성 및 이를 물리 엔진의 입력값(외력(Force), 하중 위치, 가구의 스펙 등)과 실시간 연동.
- **데이터 판독기(Readout) 업데이트**: 실시간 무게중심(Center of Mass) 상태나 넘어짐 여부, 가해지는 힘의 크기 등을 게이지나 깔끔한 타이포그래피 수치로 중앙 집중적으로 표시.

## Verification Plan

### Manual Verification
1. 변경 사항 적용 후 프론트엔드 환경을 로컬에서 실행(또는 브라우저로 엽니다).
2. **시각적 검증**: 레퍼런스 이미지와 유사한 미니멀하고 고급스러운 느낌의 UI가 렌더링되었는지 확인합니다.
3. **상호작용 검증 (Test Control Flow)**: 새로 추가된 다이얼/노브를 마우스로 조작했을 때, 값이 부드럽게 변경되며 시뮬레이터(3D 뷰) 내의 물리적 상태(예: 기울어짐 변화, 힘의 크기 등)가 즉각적으로 반영되는지 확인합니다.
4. **데이터 표시 검증**: 가구가 넘어지거나 불안정한 상태가 될 때 패널의 데이터 게이지(ex. Stability %)나 수치들이 정확하게 연동되어 변경되는지 확인합니다.
